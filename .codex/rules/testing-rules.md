# Testing Rules

## Prioridades

- Testar regras de status de figurinha.
- Testar services com comportamento critico.
- Testar formatacao de listas compartilhaveis.
- Testar fluxos principais quando a estrutura de teste existir.

## Padrao

- Testes devem ser focados e legiveis.
- Evitar mocks excessivos quando uma funcao pura resolve.
- Bugs corrigidos devem ganhar teste quando possivel.

## Validacao

Antes de avancar fase, rodar:
- Typecheck, se existir.
- Lint, se existir.
- Testes, se existirem.
