# Global Rules

## Rotina obrigatoria

1. Ler os arquivos base indicados no README antes de iniciar.
2. Identificar a fase atual.
3. Escrever um plano curto antes de implementar.
4. Fazer mudancas pequenas, coesas e alinhadas com a fase.
5. Validar com typecheck, lint e testes quando existirem.
6. Atualizar memoria local ao terminar.

## Restricoes

- Nao implementar funcionalidades fora da fase atual sem pedido explicito.
- Nao instalar dependencias sem justificativa.
- Nao colocar secrets no repositorio.
- Nao criar codigo solto ou duplicado sem necessidade.
- Nao sobrescrever arquivos importantes sem entender o impacto.
- Nao alterar arquitetura ou padrao visual sem registrar decisao.

## Documentacao viva

Toda informacao importante para continuidade deve ser registrada em `.codex/memory/`. Nao depender de memoria externa.
