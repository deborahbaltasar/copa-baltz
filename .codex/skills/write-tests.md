# Skill: Write Tests

## Objetivo

Adicionar testes relevantes para regras e fluxos.

## Passos

1. Identificar comportamento critico.
2. Preferir testes de funcoes puras quando possivel.
3. Cobrir casos principais e limites.
4. Manter testes legiveis.
5. Rodar suite.

## Alvos iniciais

- `stickerStatus`.
- Calculo de progresso.
- Geracao de listas de faltantes e repetidas.
- Services com mocks simples.
